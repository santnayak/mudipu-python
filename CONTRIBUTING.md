# Contributing to Mudipu 🎉

Hey there! Thanks for wanting to contribute to Mudipu. We're excited to have you here!

## Code of Conduct

Be nice, be respectful, and let's build something awesome together! 🙌

## Found a Bug? 🐛

Create an issue and explain the bug! Include:

- What you expected to happen
- What actually happened
- Steps to reproduce it
- Your environment (OS, Python version, etc.)

That's it! We'll take a look and work on fixing it together.

## Have a New Feature Idea? 💡

Create an issue and let's discuss it! Tell us:

- What feature you'd like to see
- Why it would be useful
- Any ideas on how it might work

We'll chat about it, refine the idea, and then you can implement it (or we can help!).

## Want to Contribute Code? 🚀

### Branching Strategy

We keep it simple:

- **`main` branch** → Stable, production-ready code
- **`development` branch** → All PRs get merged here first, then versioned into main
- **`feature/your-feature-name`** → Your feature branch for development

### Workflow

1. Fork the repo
2. Create a feature branch from `development`:
   ```bash
   git checkout development
   git checkout -b feature/awesome-feature
   ```
3. Make your changes
4. Create a PR to merge into `development`
5. We'll review it, discuss if needed, and merge!

### Before You Submit

- Make sure your code works
- Add tests if you can
- Do some quick code check with *black* and *ruff*
- Update docs if you're changing behavior
- Keep it simple and clean

That's it! No complicated rules, just good code and good vibes.

## Questions?

- Open a GitHub issue
- Start a discussion
